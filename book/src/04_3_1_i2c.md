âœ… Why does passing the same IÂ²C instance to two sensors not work, despite both being on the same IÂ²C bus?

<details>
  <summary>Answer</summary>

This is an ownership issue. Every place in memory needs to be owned by something. If we pass the IÂ²C bus to the SHTC3, the sensor owns the IÂ²C bus. It can't be owned by another sensor. Borrowing is also not possible, because the IÂ²C bus needs to be mutable. Both sensors need to be able to change it. We solve this problem by introducing a bus manager, that creates a number of proxies of the IÂ²C bus. These proxies can than be owned by the respective sensors.
</details>
